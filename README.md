# sky-media-article-creator-skill

一个面向中文自媒体创作者（尤其是公众号、头条号等以文字为主的平台）的写作 Skill。它帮助你从“主题 / 大纲”出发，按「故事 + 观点」的结构，生成口语化、有个人感、尽量没有明显 AI 味道的文章，并可选配套生成封面图和正文插图（落盘为图片 + 纯文本稿）。

## 功能概览

- 支持从主题或大纲出发，生成 1500–2000 字、适合公众号的中文文章：
  - 开头用具体故事或场景切入；
  - 中间拆解观点、方法或经验；
  - 结尾用简洁要点收束，而不是长篇鸡汤。
- 内置多种文风预设，可在使用时选择：
  - 感性故事型、干货理性型、职场专业型、轻松吐槽型。
- 默认输出为纯文本（不带 Markdown 语法），方便直接粘贴到公众号、头条号等编辑器。
- 支持生成封面图和插图：
  - 封面：带中文主题文字 + 与文章主题匹配的背景与图形元素（禁止人物 / 人脸）；
  - 插图：在文章中用结构化占位符标记插图位置，并将图片文件落盘到统一目录。

## 目录结构

- `SKILL.md`：Skill 的主说明文档，包含触发条件、交互流程、写作规范和图片生成规范。
- `agents/openai.yaml`：用于集成到 Codex / OpenAI 相关 UI 的元数据（展示名称、说明等）。
- `scripts/generate_article_assets.py`：根据文章中的 `<!-- IMAGE: ... -->` 标记，调用外部图片 API 生成封面和插图，并把纯文本文章与图片统一落盘。
- `references/personal_tone_examples.md`：用于存放你的个人语气和风格示例，写作前可参考，以便生成更贴近你本人风格的文章。

## 图片生成脚本使用方式

1. 准备一篇带 IMAGE 标记的文章（推荐纯文本），例如：

   ```text
   <!-- IMAGE: cover | prompt=画面背景是抽象的科技线条和芯片图形，前景用大号中文写着“AI 初创公司”，画面中不要出现人物或人脸 | aspect=16:9 -->

   这里是正文……

   <!-- IMAGE: inline | name=两类“中间层”产品示意 | prompt=一张信息图：左边是一层包装在大模型上方的 UI，右边是“AI 工具超市”，中间用箭头指向“容易被大厂替代”，风格简洁清晰 | aspect=16:9 -->
   ```

2. 在环境中配置图片 API 所需的变量（以 DashScope 为例）：

   ```bash
   export ALI_DASHSCOPE_API_KEY="你的真实 key"
   ```

3. 在仓库根目录运行：

   ```bash
   python scripts/generate_article_assets.py \
     --article-file /path/to/article.txt \
     --output-dir /path/to/output/folder \
     --cover-aspect 16:9
   ```

   运行完成后，输出目录中会包含：

   - `article_with_images.txt`：已经把 IMAGE 标记替换为结构化的插图占位符，例如：  
     `【插图：名称=两类“中间层”产品示意；文件=inline_2.png】`  
   - `cover.png`：封面图片（如果有封面标记）；
   - `inline_1.png`, `inline_2.png`…：正文插图。

## 个人语气与风格沉淀

你可以在 `references/personal_tone_examples.md` 中，按类型整理自己满意的文章片段（比如工具介绍、经验复盘、成长故事），并写上希望保留的语气特征。写作时参考这些示例，可以让文章更像“你自己写的”，而不是千篇一律的 AI 腔。

## 适用场景

- 需要经常写公众号/头条号/小红书长文，但不想每次从空白页开始；
- 希望文章结构稳定（故事 + 观点），语言口语化、有细节、不太“AI 味”；
- 希望一键拿到：文章正文 + 封面图 + 插图文件夹，方便直接在自媒体后台排版发布。
